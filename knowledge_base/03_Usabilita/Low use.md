# Low use

> [!info] Definizione
> Situazione in cui una parte rilevante delle funzioni disponibili in un sistema è poco conosciuta, poco usata o mai usata dagli utenti.

## Spiegazione

Negli appunti il low use è collegato a multifunzionalità e complessità d'uso: offrire molte opzioni non significa aumentare l'utilità reale, ma può disorientare l'utente.

## Dettagli importanti

Per misurarlo si ricorre all'**Use Index** (Indice di utilizzo):
`Use Index = (Numero di funzioni effettivamente utilizzate) / (Numero di funzioni disponibili)`
Un Use Index molto basso è sintomo di eccessiva complessità strutturale/funzionale rispetto ai task reali.

## Esempio

> [!example] Esempio
> Un word processor molto ricco di funzioni viene usato dal 90% degli utenti solo per i comandi base di formattazione del testo, risultando in un low use per le funzioni avanzate.
